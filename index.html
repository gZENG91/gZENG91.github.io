3.php中传值与传引用的区别?

按引用传递：函数范围内对值的任何改变在函数外部也能反映出这些修改

4.strlen()与mb_strlen的作用与区别?

strlen无法正确处理中文字符串,对于GB2312的中文编码，strlen得到的值是汉字个数的2倍，而对于UTF-8编码的中文，就是3倍（在 UTF-8编码下，一个汉字占3个字节）
采用mb_strlen函数可以较好地解决这个问题

5.isset() 、empty()与is_null的区别?

(1)当变量未定义时，is_null() 和“参数本身”是不允许作为参数判断的，会报Notice警告错误；
(2)empty , isset首先都会检查变量是否存在，然后对变量值进行检测。而is_null 和 “参数本身”只是直接检查变量值，是否为null，因此如果变量未定义就会出现错误！
(3)isset()：仅当null和未定义，返回false；
(4)empty()：””、0、”0”、NULL、FALSE、array(),未定义，均返回true；
(5)is_null()：仅判断是否为null，未定义报警告；
(6)变量本身作为参数，与empty()一致，但接受未定义变量时，报警告；

6.表单中 get与post提交方法的区别?

(1)get主要从服务器上获取数据，post像服务器传送数据;
(2)get通过URL传参，在URL可以直接看到，post以请求实体提交，用户不可见
(3)get传递的数据量不能超过2KB,post一般不受限制
(4)get相对post的安全性低
(5)FORM表单提交，不填写method，默认以get方式提交

7.session与cookie的区别?

cookie：一种通过服务器设置，将数据保存到浏览器端的技术，
当再次访问时又会将保存在浏览器端的cookie数据带回给服务器；

  • 可以通过header函数来设置协议项（set-cookie:名=值）；
  • 通过函数setcookie()设置，该函数有多个参数可以设置；
    ①参数1，参数2，是设置cookie的名与值（可以不设）；
    ②参数3，设置cookie的生命周期（默认会话关闭，失效）；
    ③参数4，用户设置cookie的作用范围（默认cookie是只能在同级目录即子目录才可以共享）；
    ④参数5，可以设置cookie实现跨域共享（默认不能实现），前提是保证有相同的一级域名;

SESSION：一种将数据保存在服务器中，实现跨脚本共享数据的会话技术；

  • session有两种方式开启：
    ①：手动开启，每次要使用session必须使用函数session_start激活session机制；
    ②：自动开启（通过设置php.ini文件，默认是关闭），每次使用不用激活；不建议使用（会造成，在不跨脚本都会额外创建文件）
  • 默认是保存在cookie中；sessin中保存的sessionID保存方式：
  • 可以修改配置文件保存到URL中（当用户禁用cookie时，可以使用这个方法）
  • sessionID的生命周期：默认是会话结束，即使被失效；
  • sessionID的作用范围：默认是整站（/）；
  • sessionID默认不支持跨域；
  • session文件默认的最大生命周期（过期）是24分钟；

8.Http常见状态码

  • 2XX 成功
    200 OK，表示从客户端发来的请求在服务器端被正确处理
  • 3XX 重定向
    301 永久性重定向，表示资源已被分配了新的 URL
    302 临时性重定向，表示资源临时被分配了新的 URL
  • 4XX 客户端错误
    400 错误的请求
    401 表示发送的请求需要有通过 HTTP 认证的认证信息
    403 表示对请求资源的访问被服务器拒绝
    404 表示在服务器上没有找到请求的资源
  • 5XX 服务器错误
    500 内部服务器错误
    503 服务器目前无法为请求提供服务，但过一段时间就可以恢复服务

12.PHP缓存技术有哪些? tp是局部还是完全缓存?

(1)全页面静态化缓存，也就是将页面全部生成html静态页面，用户访问时直接访问的静态页面，而不会去走php服务器解析的流程
(2)页面部分缓存，将一个页面中不经常变的部分进行静态缓存，而经常变化的块不缓存，最后组装在一起显示
(3)数据缓存，通过一个id进行请求的数据,将数据缓存到一个php文件中,id和文件是对应的,下次通过这个id进行请求时 直接读php文件
(4)查询缓存，和数据缓存差不多,根据查询语句进行缓存;
(5)常用的缓存技术有：redis和memcache

13.session共享问题解决方案?

(1)使用数据库保存session， 使用数据库来保存session，就算服务器宕机了也没事，session照样在。
问题：程序需要定制；每次请求都进行数据库读写开销不小，另外数据库是一个单点，可以做数据库的hash来解 决这个问题。
(2)使用 memcached来保存session， 这种方式跟数据库类似，内存存取性能比数据库好很多。
问题：程序需要定制，增加了工作量；存入memcached中的数据都需要序列化，效率较低，断电或者重启电脑容易丢失数据；
(3)通过加密的cookie，在A服务器上登录，在用户的浏览器上添加加密的cookie，当用户访问B服务器时，检查有无Session，如果没有，就检验Cookie是否有效，Cookie有效的话就在B服务器上重建session。简单，高效， 服务器的压力减小了，因为session数据不存在服务器磁盘上。根本就不会出现session读取不到的问题。
问题：网络请求占用很多。每次请求时，客户端都要通过cookie发送session数据给服务器，session中数据不能太多，浏览器对cookie 的大小存在限制。不适合高访问量的情况

使用phpize脚本安装php扩展模块的好处：在安装php时没有安装的扩展，可以在以后随时安装上，且不需要重新安装PHP。

安装步骤：

切换到扩展模块目录

在php源码包被解压后的目录中有个ext子目录，这里有近70多个主流的php扩展模块安装包。
如现在要安装memcache扩展，则切换到memcache目录：
cd /software/php-5.2.14/ext/memcache

在memcache目录中执行phpize脚本

/usr/local/php/bin/phpize

执行成功会返回几行数据：

开始编译(注意–with-php-config参数)

在php.ini中找到extension_dir字段，把值修改成：

/usr/local/php/lib/php/extensions/no-debug-non-zts-20060613/

7、再在php.ini的Dynamic Extensions节位置下添加

搭建Git服务器

发表于 2017-02-15 | 分类于 Linux | | 阅读次数

先安装git

[root@localhost ~]# yum install -y git

创建一个git用户，用来运行git服务

[root@localhost ~]# adduser git

选定一个目录作为Git仓库

这里是/data/git/project.git。

执行以上命令Git命令，会创建一个裸仓库，裸仓库没有工作区，因为服务器上的Git仓库纯粹是为了共享所以不让用户直接登录到服务器上去改工作区，并且服务器上的Git仓库通常都以.git结尾。然后，把owner改为git

[root@localhost git]# chown -R git.git project.git

在/home/git/.ssh家目录中生成一个.ssh的文件，然后创建authorized_keys用来保存客服端的公钥，这样客服端登录就不用密码登录了；
[root@localhost git] touch authorized_keys

这样git服务器就搭建完成了，就可以通过

$ git clone git@serverIp:/data/git/project.git

这里就不说客户端的创建了，比较简单，当客户端安装后，将生成的公钥放到/home/git/.ssh/authorized_keys中就可以了

这里有个问题，git服务器安装后，客户端公钥也上传了，但每个git的时候都还需要输入密码，这时还需要更改/home/git/.ssh/authorized_keys的权限

[root@localhost git] chmod 644 /home/git/.ssh/authorized_keys
